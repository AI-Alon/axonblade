AxonBlade Docs

Everything you need to write, run, and understand AxonBlade programs.

Installation

Requires Python 3.11 or newer. Clone the repo and install from source:

git clone https://github.com/AI-Alon/axonblade.git
cd axonblade
pip install -e .

This installs the ablade CLI command. Verify with:

ablade version
If ablade is not found after install, add the pip scripts directory to your PATH:

Linux / macOS:
export PATH="$(python3 -m site --user-base)/bin:$PATH"

Windows (PowerShell):
$env:PATH += ";" + (python -m site --user-site).Replace("site-packages","Scripts")

First Program

Create a file called hello.axb:

>> name = "World"
write(-*cyan*- + "Hello, &{name}!" + -*reset*-)

Run it:

ablade run hello.axb

Running Files

ablade run <file.axb>          # run a program
ablade repl                    # start interactive REPL
ablade fmt <file.axb>          # print formatted source
ablade fmt --in-place <file>   # overwrite file with formatted output
ablade fmt --check <file>      # exit 1 if file would change (CI)
ablade lint <file.axb>         # run static analysis
ablade test [dir]              # discover and run *_test.axb files
ablade version                 # print version

REPL

Start an interactive session with ablade repl. Each line is evaluated immediately.

For multi-line blocks, open the block with +/ — the REPL will keep reading lines until you close it with ECB:

axb> >> x = 10
axb> write(x)
10
axb> bladeFN double(n) +/
...>     return n * 2
...> ECB
axb> write(double(x))
20
axb> EXIT

Type EXIT to quit the REPL.

Variables

Use >> to declare a new variable. Re-assign without it.

>> x = 10
x = x + 5   # re-assign
write(x)    # 15
Assigning to an undeclared name raises AxonNameError. Always declare with >> first.

Strings

String literals use double quotes. Embed any expression with &{} — variables, arithmetic, and function calls all work:

>> n = 42
write("The answer is &{n}")        # The answer is 42
write("2 + 2 = &{2 + 2}")          # 2 + 2 = 4
>> a = 3
>> b = 4
write("&{a} * &{b} = &{a * b}")   # 3 * 4 = 12

Concatenate with +:

>> s = "Hello" + ", " + "World"   # Hello, World

Escape sequences inside strings:

SequenceMeaning
\nNewline
\tTab
\"Double quote
\\Backslash

Comments use # for single-line and #/ ... /# for multi-line:

# this is a single-line comment

#/
  this spans
  multiple lines
/#

ECB (End Code Block) closes every block — functions, loops, conditionals, and classes all end with ECB.

Pipeline

The |> operator passes the left-hand value as the first argument to the right-hand function call.

bladeFN double(x#int) +/ return x * 2 ECB
bladeFN inc(x#int) +/ return x + 1 ECB
bladeFN square(x#int) +/ return x * x ECB

# 3 → double → inc → square = 49
>> result = 3 |> double() |> inc() |> square()
write(result)   # 49

Control Flow

if / elif / else

if x > 0 +/
    write("positive")
ECB elif x == 0 +/
    write("zero")
ECB else +/
    write("negative")
ECB

while

>> i = 0
while i < 5 +/
    write(i)    # prints 0 1 2 3 4
    i = i + 1
ECB

for

for item in [1, 2, 3] +/
    write(item)   # prints 1 2 3
ECB

# range(n) → [0, 1, ..., n-1]
for i in range(3) +/
    write(i)   # 0  1  2
ECB

# range(start, end) → [start, ..., end-1]
for i in range(5, 8) +/
    write(i)   # 5  6  7
ECB

Lists

Lists are ordered collections. Create them with square brackets.

>> nums = [1, 2, 3, 4]
write(len(nums))   # 4

Index with ~i~ (tilde syntax). Negative indices count from the end:

write(nums~0~)    # 1  (first)
write(nums~3~)    # 4  (last)
write(nums~-1~)   # 4  (last, negative index)
write(nums~-2~)   # 3

Slice with ~start:end~ — either bound can be omitted:

>> letters = ["a", "b", "c", "d", "e"]
write(letters~1:3~)   # [b, c]
write(letters~:2~)    # [a, b]
write(letters~3:~)    # [d, e]

Slicing works on strings too:

>> s = "hello"
write(s~1:4~)   # ell

List methods:

OperationDescription
lst.append(val)Add to end
lst.pop()Remove and return last item
lst.insert(i, val)Insert at index
lst.remove(val)Remove first occurrence
lst.reverse()Reverse in place
lst.copy()Return a shallow copy
len(lst)Number of items
>> items = ["a", "b", "c"]
items.append("d")
write(items~3~)   # d
items.reverse()
write(items~0~)   # d

Dictionaries

Dictionaries store key-value pairs. Keys are unquoted identifiers at creation, but quoted strings when accessing with ~:

>> person = {name: "Alon", age: 17}
write(person~"name"~)   # Alon
write(person~"age"~)    # 17

Assign new keys or update existing ones:

person~"age"~ = 18
person~"city"~ = "Tel Aviv"

Built-in dictionary methods:

MethodDescriptionExample
dict.keys()List of all keysperson.keys() → ["name", "age"]
dict.values()List of all valuesperson.values() → ["Alon", 17]
dict.items()List of [key, value] pairsperson.items() → [["name","Alon"],...]
dict.get(key, default)Get value or default if key missingperson.get("phone", "N/A") → N/A

Iterate over a dict using keys():

for k in person.keys() +/
    write("&{k}: &{person~k~}")
ECB
# name: Alon
# age: 17

Functions

bladeFN greet(name#str) +/
    write("Hello, &{name}!")
ECB

greet("Alon")

Functions are values — pass them around, return them, or define them inline:

>> double = bladeFN(x) +/ return x * 2 ECB

Closures capture the enclosing scope at definition time:

bladeFN make_adder(n) +/
    return bladeFN(x) +/
        return x + n
    ECB
ECB

>> add5 = make_adder(5)
write(add5(3))   # 8

Classes

Use bladeGRP to define a class. The instance reference is blade (not self).

bladeGRP Point +/
    bladeFN init(blade, x#int, y#int) +/
        blade.x = x
        blade.y = y
    ECB
    bladeFN to_str(blade) +/
        return "(&{blade.x}, &{blade.y})"
    ECB
ECB

>> p = Point(3, 4)
write(p.to_str())

Type Annotations

Add #type after a parameter name to enforce the type at call time. Unannotated parameters accept any type.

AnnotationType checked

count#intinteger (whole number)

items#listlist

    for i in range(times) +/
        write("Hello, &{name}!")
    ECB

greet("Alon", 2)   # Hello, Alon! (×2)

Error Handling

Use try / catch to handle runtime errors. The caught value is a dict with type, message, and line:

try +/
    >> x = 10 / 0
ECB catch e +/
    write(e~"type"~)      # AxonDivisionError
    write(e~"message"~)   # division by zero
    write(e~"line"~)      # 2
ECB

Raise your own errors with raise:

bladeFN divide(a#int, b#int) +/
    if b == 0 +/
        raise "cannot divide by zero"
    ECB
    return a / b
ECB

try +/
    write(divide(10, 0))
ECB catch e +/
    write(-*red*- + "Error: &{e}" + -*reset*-)   # Error: cannot divide by zero
ECB

Grid System

Create a grid with grid(cols, rows), then call methods on the returned object:

>> g = grid(20, 10)   # 20 columns, 10 rows
MethodDescription
g.set(x, y, color)Set tile background color
g.get(x, y)Get tile ANSI color code
g.fill(color)Fill entire grid with color
g.set_char(x, y, ch)Set character on tile
g.get_char(x, y)Get character on tile
g.clear()Reset all tiles to black
g.loop(fn, fps)Start game loop calling fn at given fps
g.on_key(key, fn)Register key press handler
g.stop()Stop the game loop
g.width()Number of columns
g.height()Number of rows
g.render()Manually render the grid to terminal

Example — a simple moving dot:

>> g = grid(20, 10)
>> x = 0
>> dx = 1

bladeFN update() +/
    g.fill(-*black*-)
    x = x + dx
    if x >= 19 -o x <= 0 +/
        dx = dx * -1
    ECB
    g.set(x, 5, -*cyan*-)
ECB

g.on_key("q", bladeFN() +/ g.stop() ECB)
g.loop(update, 10)   # 10 fps, press q to quit

String Dot-Methods

Call methods directly on any string value or variable using dot notation. These work on string literals, variables, and intermediate expressions.

MethodDescriptionExample
s.upper()Uppercase"hello".upper() → HELLO
s.lower()Lowercase"HELLO".lower() → hello
s.strip()Remove leading/trailing whitespace" hi ".strip() → hi
s.trim()Alias for strip()" hi ".trim() → hi
s.split(sep)Split into list on separator"a,b,c".split(",") → [a, b, c]
s.replace(old, new)Replace all occurrences"hi world".replace("world","Axon")

Methods can be chained — each call returns a new string:

>> result = "  axonblade  ".strip().upper()
write(result)   # AXONBLADE

>> parts = "one,two,three".split(",")
write(parts)   # [one, two, three]

write("Hello".starts_with("He"))   # true
write("Hello".ends_with("lo"))     # true
write("Hello".contains("ell"))     # true

Formatter — ablade fmt

The formatter rewrites an AxonBlade source file to a canonical style — consistent indentation, spacing around operators, blank lines between declarations, and no trailing whitespace. It works by parsing the file into an AST and re-emitting it, so the output is always syntactically valid.

ablade fmt hello.axb             # print formatted source to stdout
ablade fmt --in-place hello.axb  # overwrite the file
ablade fmt --check hello.axb     # exit 1 if the file would change (use in CI)

Formatting rules

RuleDetail
Indentation4 spaces per nesting level. Tabs are never used.
Binary operatorsExactly one space on each side: x + 1, a == b
Top-level declarationsOne blank line before each bladeFN or bladeGRP
Trailing whitespaceStripped from every line
Final newlineAlways present

Linter — ablade lint

The linter performs static analysis on a source file without running it. It reports errors and warnings in a machine-readable format and exits with a code that CI scripts can act on.

ablade lint hello.axb

Output format

hello.axb:12:5: [error] undefined variable 'x'
hello.axb:8:1: [warning] variable 'tmp' is declared but never read

Exit codes

CodeMeaning
0No errors or warnings
1At least one error
2At least one warning, no errors

Diagnostics

LevelDiagnostic
[error]Identifier used before any declaration in scope
[error]Function called with the wrong number of arguments (at known call sites)
[warning]Variable declared with >> but never read
[warning]Unreachable statements after return in a function body
[warning]Variable name shadows a variable from an outer scope

Test Runner — ablade test

The test runner discovers all files named *_test.axb under the given directory (default: .), runs each one through the evaluator, and reports pass/fail results per test() call.

ablade test           # search from current directory
ablade test tests/    # search from a specific directory

Test builtins

BuiltinDescription
test(name, fn)Register and run a named test case. fn is a zero-argument bladeFN.
assert_eq(a, b)Fail with a diff if a != b
assert_true(val)Fail if val is falsy
assert_raises(fn)Fail if calling fn() does not raise an error

Example test file (math_test.axb)

test("addition", bladeFN() +/
    assert_eq(1 + 1, 2)
    assert_eq(10 + 32, 42)
ECB)

test("division by non-zero", bladeFN() +/
    assert_eq(10 / 2, 5)
ECB)

test("division by zero raises", bladeFN() +/
    assert_raises(bladeFN() +/
        >> x = 1 / 0
    ECB)
ECB)

Output

  ✓  tests/math_test.axb :: addition
  ✓  tests/math_test.axb :: division by non-zero
  ✓  tests/math_test.axb :: division by zero raises

  ✓ 3 passed   ✗ 0 failed

Bytecode & Performance

AxonBlade V2 compiles source files to compact bytecode before executing them. Instead of walking the AST on every run, the compiler emits a flat sequence of opcodes that a fast stack-based virtual machine (VM) dispatches in a tight loop.

How it works

  1. The compiler traverses the AST and emits Instruction objects, each carrying an Opcode and an optional argument (a constant index, a variable name, or a jump target).
  2. The output is a CodeObject — a self-contained bundle holding the instruction list, a constants table, a names table, and metadata for parameters and captured upvalues.
  3. The VM executes CodeObjects by maintaining a value stack and a call-frame stack. Each function call pushes a new frame; RETURN pops it.

Opcode groups

GroupOpcodes
StackPUSH_CONST, PUSH_NULL, PUSH_TRUE, PUSH_FALSE, POP
VariablesLOAD_VAR, DEFINE_VAR, STORE_VAR
UpvaluesLOAD_DEREF, STORE_DEREF — for closed-over variables
ArithmeticADD, SUB, MUL, DIV, MOD, POW, NEG
ComparisonEQ, NEQ, LT, GT, LTE, GTE
LogicAND, OR, NOT
JumpsJUMP, JUMP_IF_FALSE, JUMP_IF_TRUE
CollectionsMAKE_LIST, MAKE_DICT, GET_INDEX, SET_INDEX, GET_SLICE
AttributesGET_ATTR, SET_ATTR
FunctionsMAKE_FN, CALL, RETURN
ClassesMAKE_CLASS
StringsBUILD_FSTRING
ExceptionsSETUP_TRY, POP_TRY, RAISE
ModulesIMPORT
