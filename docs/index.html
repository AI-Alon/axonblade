Why AxonBlade

Everything you need, nothing you don't

A minimal yet expressive language designed for scripting, terminal programs, and interactive grid-based applications.

🎨

Color Literals

First-class color values like -*red*- and -*cyan*- produce ANSI codes directly — no escape sequences, no imports.

🟦

Native Grid Primitive

grid(cols, rows) creates a live terminal canvas. Set tiles, fill regions, attach a game loop — all in the language itself.

⚡

Clean Syntax

Readable keywords (bladeFN, bladeGRP), pipeline operator |>, and string interpolation &{expr} keep code expressive.

🔒

Type Annotations

Optional inline parameter types like name#str and count#int are checked at call time — safe without verbosity.

🔗

Closures & Classes

Functions are first-class, closures capture lexical scope, and bladeGRP provides a clean OOP model with fields and methods.

📦

Module System

Import any .axb file with uselib -name-. Built-in math and string modules are included.

🎨

VSCode Extension

Syntax highlighting for every AxonBlade token — keywords, color literals, string interpolation, type annotations, and more. Bracket matching and auto-indent included.

📚

Expanded Stdlib

Six new built-in modules: io, json, http, regex, datetime, and random. Plus string dot-methods like "hello".upper().

Get Started

Up and running in seconds

Clone the repo, install with pip, write your first program, and run it.

  1. Install AxonBlade

    Requires Python 3.11+. Clone the repo and install with pip — the ablade CLI is ready.

  2. Write your first program

    Create hello.axb and add a few lines of AxonBlade code.

  3. Run it

    Execute your file from the terminal with the ablade run command.

  4. Explore the docs

    Read the full language reference, check out examples, and build something cool.

Language at a Glance

Familiar ideas, fresh syntax

Browse code snippets across the core features of AxonBlade.

variables.axb
# Declare with >>, re-assign without >> score = 0 >> name = "player" >> alive = true score = score + 10 # F-string interpolation with &{} write("&{name} scored &{score}") # Color literals >> label = -*yellow*- + "WARNING" + -*reset*- write(label)
functions.axb
# Type annotations on parameters bladeFN add(a#int, b#int) +/ return a + b ECB # Closures capture lexical scope bladeFN make_counter() +/ >> n = 0 return bladeFN() +/ n = n + 1 return n ECB ECB >> tick = make_counter() write(tick()) # 1 write(tick()) # 2
classes.axb
bladeGRP Animal +/ bladeFN init(name#str, hp#int) +/ blade.name = name blade.hp = hp ECB bladeFN damage(amount#int) +/ blade.hp = blade.hp - amount write("&{blade.name} hp: &{blade.hp}") ECB ECB >> wolf = Animal("Wolf", 100) wolf.damage(25)
grid_demo.axb
# Create a 20x10 terminal grid >> g = grid(20, 10) g.fill(-*blue*-) # Draw a red square >> x = 0 while x < 5 +/ >> y = 0 while y < 5 +/ g.set(x, y, -*red*-) y = y + 1 ECB x = x + 1 ECB # Start game loop at 10 fps g.loop(bladeFN() +/ # update logic here ECB, 10)
pipeline.axb
# Pipeline operator |> chains calls bladeFN double(x) +/ return x * 2 ECB bladeFN add_one(x) +/ return x + 1 ECB # 5 |> double() |> add_one() = 11 >> result = 5 |> double() |> add_one() write(result) # 11 # uselib to import modules uselib -math- >> r = 9 |> math.sqrt() write(r) # 3.0

Expressive by design

AxonBlade was built to feel natural for scripting and visual programs. Every construct is intentional — the syntax encourages clarity without being verbose.

  • ✅  Declare once with >>, reassign freely
  • ✅  Color literals are just values — no setup needed
  • ✅  Anonymous functions inline anywhere
  • ✅  blade keyword replaces self — cleaner classes
  • ✅  Try/catch, raise, error types built-in
  • ✅  Module system with uselib
  • ✅  REPL for interactive exploration
